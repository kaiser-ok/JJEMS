執行摘要
核心主張與主要發現
核心主張

現有大多數商業 EMS 的調度目標是「當日能源收益最大化」，卻系統性地忽略了每次充放電對電池壽命的磨耗成本。本白皮書提出的 Battery Lifetime-Aware EMS 方法論，將電池衰退成本、故障風險成本納入即時調度的目標函數，使 EMS 從短期套利工具升級為長期資產價值優化器。

傳統 EMS 只問一個問題：「現在放電有沒有錢賺？」
Battery Lifetime-Aware EMS 問的是：「現在放電的收益，是否大於這次放電造成的循環老化、熱應力、RUL 損失與故障風險？」

本文件涵蓋以下內容：

  • 三層控制架構設計（BMS 硬限制 / 健康感知軟約束 / 經濟優化）
  • 以實測 LFP 老化參數為基礎的衰退成本模型
  • Rack 級異質健康狀態下的功率分配方法
  • 四層驗證路徑（文獻驗證 → 回溯分析 → 加速老化測試 → 案場 A/B 測試）
  • 分三階段的導入框架（監控 → 建議 → 閉環）

2.2 三層控制政策

Layer 1 — BMS 硬安全約束（不可被 EMS 覆蓋）

來自 BMS / PCS / 保護系統，代表物理邊界，任何情況下不得違反：

Vmin ≤ Vcell ≤ Vmax　// 728V – 949V (1P260S) |I| ≤ Imax　// 最大電流 Tcell ≤ Tmax　// 電芯最高溫度 ≤ 45°C (→ 停機) SoC ∈ [SoCmin, SoCmax]　// 硬邊界 10%–95%
Layer 2 — 健康感知軟約束（Battery Intelligence 輸出）

來自 SoH、RUL、風險模型，為 EMS Optimizer 的軟邊界，在 Layer 1 允許範圍內動態收緊：

SoC ∈ [SoCrec,min, SoCrec,max]　// 依 SoH 動態調整 C-rate ≤ Crec,max　// 依溫度、SoH 降額 CycleCountdaily ≤ Nbudget　// 每日循環預算 Prack,i ≤ Pmax,i(SoHi, Ti, Riski)　// Rack 級差異化限制
Layer 3 — 經濟優化目標（EMS Optimizer）

在 Layer 1 和 Layer 2 約束下求解最優調度：

max Σt [ Revenue(t) − Cdegrad(t) − Crisk(t) ] subject to: Layer 1 constraints (hard, non-negotiable) Layer 2 constraints (soft, health-aware) Grid code requirements Contract capacity obligations
3
衰退成本模型
將電池壽命損耗量化為每 kWh 的財務成本

3.1 循環替換成本基準

對於本案場的 LFP 系統（CAPEX NT$9,520,000 / 476 kWh / 6,000 次循環壽命），每 kWh 放電對應的資本消耗（循環替換成本基準）為：

Cbase = CAPEX / (Enominal × Ncycle,life) = 9,520,000 / (476 × 6,000) ≈ 3.33 NT$/kWh

這是靜態基準值；實際每次放電的衰退成本，受三個動態修正係數影響。

3.2 動態修正係數

溫度修正係數 ftemp

LFP 電池在 25°C 時老化速率最低；溫度每偏離最佳點 1°C，老化速率約增加 1.2%：

ftemp = 1 + max(0, |Tcell − 25°C|) × 0.012 // 當前工況：T = 29.4°C → f_temp = 1 + 4.4 × 0.012 = 1.053

依據：Arrhenius 模型，與 EVE LF280K 公開老化數據吻合。

C-rate 修正係數 fcrate

LFP 在 0.3C 以下老化最緩；高倍率放電加速活性物質損失：

fcrate = 1 + max(0, Crate − 0.3) × 0.08 // 當前工況：C-rate = 0.42 → f_crate = 1 + 0.12 × 0.08 = 1.010
SoH 修正係數 fsoh

已退化的電池，相同的循環操作對剩餘壽命的傷害更大（邊際衰退加速）：

fsoh = 1 + (1 − SoHfleet/100) × 0.5 // 當前艦隊平均 SoH = 97.1% → f_soh = 1 + 0.029 × 0.5 = 1.015

3.3 完整衰退成本計算

// 每次放電的衰退成本 Cdegrad = Cbase × ftemp × fcrate × fsoh × Edischarge = 3.33 × 1.053 × 1.010 × 1.015 × Edis ≈ 3.58 NT$/kWh × Edischarge // 故障風險成本（基於 Rack 風險分數） Crisk = Edischarge × Pfault,avg × Crisk,unit = Edischarge × 0.112 × 2.0 ≈ 0.22 NT$/kWh × Edischarge // 最終目標函數（每次 dispatch action） Net_value = Revenue − Cdegrad − Crisk = Revenue − (3.58 + 0.22) × Edischarge

4.2 五級風險降載矩陣

EMS 依 Battery Intelligence Layer 輸出的風險等級，採取分級主動控制：

等級觸發條件（示例）EMS 主動動作備註
正常 SoH ≥ 97%、Risk < 10% 正常調度，不限制 標準操作
觀察 SoH 94–97%、Risk 10–20% 限制 DoD 至 80%、C-rate ≤ 0.4C、提高監控頻率 預防性降額
警告 SoH 90–94%、Risk 20–35% Rack 放電優先級降低、避免高溫高功率窗口 主動降載
危急 SoH < 90%、Risk > 35% 停止該 Rack 充放電、通知維護人員、切換備援 Rack 隔離處理
緊急 熱失控前兆、多項告警同時觸發 交由 BMS/PCS/保護系統執行安全停機 BMS 最終防線
設計原則

EMS 做主動控制，BMS 做最後防線。EMS 在 BMS 告警觸發之前，就透過 Battery Intelligence Layer 的預測性分析介入，避免電池進入需要 BMS 保護性停機的狀態。緊急等級的處置權完全移交給 BMS/PCS，EMS 不得干預。

5.1 回溯驗證分析流程（Layer B 詳細說明）

Layer B 是成本最低、最快可執行的驗證方式，建議作為第一個啟動項目：

  1. 資料準備：匯出既有案場 BMS 歷史資料（建議 ≥ 6 個月）— Pack 電壓、電流、溫度、SoC，採樣間隔 ≤ 60s
  2. 風險標記：對每個時間點，計算當時的 estimateDegradationCost() 輸出，標記「高風險操作窗口」（C-rate > 0.4、T > 32°C、SoH 下降偏快）
  3. SoH 退化分析：按月計算各 Rack 的 SoH 下降斜率，確認是否和風險標記呈正相關
  4. 相關性驗證：若高風險窗口 → 後續 SoH 下降更快，代表模型預測方向正確
  5. 報告輸出：Pearson/Spearman 相關係數 + 視覺化圖表，作為技術白皮書的附件 B

6.1 為什麼毛收益下降但淨收益上升？

Battery-Aware EMS 有時會「放棄」一部分短期收益，以換取更低的衰退成本。這是反直覺的，但數學上可以證明：

  • 傳統 EMS 的毛收益較高（NT$1,541K），但衰退成本 NT$620K，淨值 NT$841K
  • Battery-Aware EMS 毛收益略低（NT$1,490K），但衰退成本降至 NT$430K，淨值 NT$1,025K
  • 差額 NT$184K/年，複利 15 年的現值效益超過 NT$1.8M（r=5%）

這正是「short-term exploitation → lifetime value optimization」的核心轉變。

7
導入路徑
三階段漸進式部署，降低客戶風險

為降低客戶對「AI 自動控制」的疑慮，並建立信任，建議採用三階段漸進式導入方式：

Phase 1
監控模式
Monitoring Only
Battery Intelligence Layer 產生的 SoH、RUL、衰退趨勢、風險分數，僅顯示在 EMS Dashboard，不影響實際調度。

目標：建立數據基線，讓 operator 熟悉指標意義，積累 Layer B 驗證所需的歷史資料。
建議期間：1–3 個月
Phase 2
建議模式
Recommendation
系統自動產生降載建議（如「今日午後建議 RACK-07 限制放電至 40kW」），由 operator 手動審批後執行。

目標：驗證建議品質，讓 operator 逐步建立信任，累積決策日誌。
建議期間：3–6 個月
Phase 3
閉環優化
Closed-Loop
EMS Optimizer 自動將 Battery Intelligence 輸出納入調度決策，包括 Rack 級差異化分配、C-rate 自動限制、高溫窗口降載。

目標：全自動 lifetime value optimization，Operator 監督即可。
